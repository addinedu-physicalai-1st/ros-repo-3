# [기술 가이드] 하이브리드 RL 자율주행 시스템 최적화 및 디버깅

본 문서는 `pinky_core`의 하이브리드 주행 시스템(Nav2 Global + RL Local)에서 발생하는 제어 및 인지 문제를 해결하고, 안정적인 자율주행을 달성하기 위한 최적화 가이드를 제공합니다.

---

## 1. 현재 시스템의 주요 문제점 진단

### 1.1 제어 불안정 (앞뒤 흔들림 및 곡선 주행)
*   **제어권 충돌:** RL 모델이 출력한 타겟 속도에 대해 `RlController`가 다시 PD 제어를 수행하면서 제어 신호가 증폭(Over-shoot)되어 진동이 발생함.
*   **가속도 제한 부재:** RL 모델의 급격한 출력 변화를 하드웨어가 따라가지 못해 물리적 진동이 발생함.
*   **직진성 부족:** 좌우 모터의 미세한 속도 편차를 RL 모델이 실시간으로 보정하지 못하거나, PD 제어기가 과도하게 개입하여 직선 주행이 불가능함.

### 1.2 조향 오류 (목표가 뒤에 있을 때 전진 문제)
*   **Lookahead 지점 선정 오류:** Nav2 경로상의 너무 먼 지점(예: 0.5m 앞)을 RL 목표로 설정하여, 로봇이 뒤로 도는 대신 전진하며 크게 회전하려 함.
*   **하드코딩된 제어 로직 간섭:** `std::abs(angle_diff) > 100도` 일 때 작동하는 강제 회전 로직이 RL 모델의 부드러운 전환을 방해하고 제어권 채터링을 유발함.

### 1.3 장애물 충돌 (Sim-to-Real Gap)
*   **인지 누락:** LiDAR 24개 섹터로 압축하는 과정에서 실제 환경의 노이즈(먼지, 유리창 등)가 섞여 장애물을 "없다"고 오판함.
*   **안전 계층의 한계:** 현재 전방 장애물 발견 시 선속도만 제어(`v=0`)하고 조향에는 개입하지 않아, 회피 기동이 불가능함.

## 2. 현 시스템의 알고리즘 및 필터 적용 현황 분석

### 2.1 경로 추종 (Pure Pursuit) 현황
*   **상태:** 전통적인 Pure Pursuit 알고리즘은 구현되어 있지 않으며, RL 모델의 목표점 선정을 위한 **'고정 Lookahead'** 방식만 존재함.
*   **문제점:** `kLookaheadDist = 0.5m` 고정값을 사용하여, 급격한 곡선이나 목표가 로봇 뒤에 있을 때 경로를 가로질러 가려는 경향이 강함. 이로 인해 로봇이 제자리 회전 대신 전진부터 수행하게 됨.
*   **개선 방향:** 속도와 곡률에 따른 **가변 Lookahead 거리** 도입이 필요함.

### 2.2 노이즈 필터 적용 현황
*   **액션 스무딩 (EMA 필터):** `kEmaAlpha = 0.4`가 적용되어 있으나, 물리적 진동을 잡기에는 필터 강도가 약함(Alpha 값이 큼). 이전 값의 60%를 유지하는 수준이라 급격한 속도 변화(Jitter)가 모터에 그대로 전달됨.
*   **LiDAR 데이터 전처리:** `NaN/Inf` 제거 및 5cm 임계값 필터만 존재함. 실제 환경의 센서 튀는 값(Spike noise)을 억제하는 **시간축 필터(Temporal/Median filter)**가 부재하여 장애물 판단이 불안정함.
*   **센서 퓨전:** `BNO055` IMU와 오도메트리를 결합한 기본적인 추정은 수행 중이나, RL 입력용 속도 피드백에 대한 **저대역 통과 필터(LPF)** 처리가 미비하여 제어 루프의 진동을 유발함.

## 3. 성공적인 주행을 위한 5대 개선 가이드

### 1단계: 내부 상태 투명성 확보 (디버깅 강화)
로봇이 "왜" 그렇게 판단했는지 블랙박스를 해소해야 합니다.
*   **관측 벡터(28D) 로깅:** `ObservationBuilder`가 생성한 28개의 수치를 실시간으로 출력하여 시뮬레이션 데이터와 대조.
*   **Lookahead Goal 시각화:** Nav2 경로에서 추출한 목표 지점을 GUI 상에 점으로 표시하여 "로봇이 어디를 보고 가는지" 확인.
*   **Action 차트화:** `raw_action`과 `final_cmd_vel`을 그래프로 비교하여 진동의 근원지 파악.

### 2단계: 목표 지점(Lookahead) 선정 최적화
*   **가변 Lookahead 거리:** 곡률이 심하거나 헤딩 차이가 클 때는 Lookahead 거리를 단축(0.15m ~ 0.2m)하여 정교한 추종 유도.
*   **순차적 경로 추종:** 로봇과 가장 가까운 경로 지점부터 순차적으로 추종하여 로봇이 경로의 형상을 정확히 인지하게 함.

### 3단계: 제어 계층 단순화 및 필터링
*   **PD 제어 비중 축소:** RL 모델의 출력을 최대한 신뢰하고, `RlController`의 `KP` 값을 낮추어(예: 0.5 이하) 제어기 간섭 최소화.
*   **EMA 필터 강화:** `kEmaAlpha`를 `0.1 ~ 0.2`로 낮추어 부드러운 가감속 유도 및 고주파 진동 억제.

### 4단계: 실기 환경 노이즈 대응
*   **LiDAR 필터링 개선:** `LidarProcessor`의 최소 거리 필터(5cm)가 장애물 감지를 방해하는지 확인하고, 시간축 필터를 추가하여 데이터 안정화.
*   **지능형 Safety Layer:** 전방 장애물 근접 시 단순히 멈추는 것이 아니라, 장애물 반대 방향으로 조향을 강제하는 로직 추가.

### 5단계: 상태 기반 주행(Behavior Tree) 도입
*   **상황별 모드 분리:**
    *   **정상 주행:** RL 모델이 글로벌 경로 추종.
    *   **제자리 회전:** 각도 차이가 클 때 RL에게 아주 짧은 거리의 목표를 주어 제자리 회전 유도.
    *   **회복(Recovery):** 충돌 위험이나 고착 상황 발생 시 Nav2에 재계획 요청 및 후진 수행.

## 3. 핵심 파라미터 튜닝 체크리스트

| 파라미터 | 권장 설정 방향 | 기대 효과 |
| :--- | :--- | :--- |
| `kKpV`, `kKpW` | 현재보다 대폭 하향 (1.5 -> 0.5) | 제어기 오버슈트 및 앞뒤 흔들림 제거 |
| `kEmaAlpha` | 낮게 설정 (0.4 -> 0.15) | 움직임의 부드러움 증대, 진동 억제 |
| `kLookaheadDist` | 상황에 따라 가변 (0.2 ~ 0.5) | 뒤쪽 목표에 대한 빠른 대응 및 제자리 회전 |
| `kAngularDeadzone` | 미세 조정 (0.05) | 직진 주행 시 불필요한 미세 조향 억제 |

## 4. 결론
성공적인 자율주행의 핵심은 **"RL 모델이 시뮬레이션과 동일한 수준의 고품질 관측 데이터를 받게 하고, 그 출력이 물리적 노이즈 없이 모터에 전달되도록 하는 것"**입니다. 시각화 디버깅을 통해 데이터 흐름을 먼저 검증한 후, 위 가이드에 따라 하향식 튜닝을 진행하십시오.

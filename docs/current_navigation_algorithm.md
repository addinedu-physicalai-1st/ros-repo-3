# 현재 로봇 주행 알고리즘 요약 (Robot Navigation Algorithm Summary)

**기준 문서:** `docs/phase5_nav_debug_enhancement_report.md` 및 `pinky_core`, `pinky_station` 소스코드 분석 결과
**작성일:** 2026-04-07

---

## 1. 시스템 아키텍처 개요
로봇의 주행은 크게 원격 명령을 내리는 **스테이션(Station)**과 실제 하드웨어 제어와 딥러닝 추론을 담당하는 **코어(Core)**로 나뉘어 동작합니다.

* **pinky_station (명령 및 전역 경로 계획)**
  * GUI 및 `NavWorker`를 통해 목표 지점을 설정하고, 필요 시 Nav2 브리지를 통해 글로벌 경로(Global Path)를 요청합니다.
  * 계산된 웨이포인트(Waypoint)를 따라 순차적으로 `NAV_GOAL` 메시지를 ZeroMQ를 통해 코어 측으로 전송하며, `NAV_CANCEL`로 정지할 수 있습니다.
* **pinky_core (지역 제어 루프 - NavLoop)**
  * `robot_app.cpp`에 구현된 `NavLoop` 스레드가 20Hz(50ms 주기)로 동작합니다.
  * 스테이션으로부터 받은 목표(Goal)를 향해 현재 Odometry와 LiDAR 센서 상태를 기반으로 모터를 직접 제어(CmdVel)합니다.

---

## 2. NavLoop의 3가지 주행 모드
`NavLoop`는 목표까지의 거리(`dist`), 각도 차이(`angle_diff`), 센서 및 모델 유무에 따라 3가지 모드를 동적으로 전환하며 동작합니다.

### 2.1 Turn-first 모드 (제자리 회전)
목표 지점의 방향이 로봇의 현재 헤딩(heading)과 크게 차이가 날 때, 주행 전 먼저 목표 방향을 바라보도록 제자리 회전하는 모드입니다.
* **히스테리시스 (채터링 방지)**: 경계 부근에서 회전과 전진을 반복하는 현상을 막기 위해 진입과 탈출 각도를 다르게 설정했습니다.
  * 진입 각도(`turn_first_enter_rad`): 105° 이상 차이 날 때
  * 탈출 각도(`turn_first_exit_rad`): 80° 이하로 좁혀졌을 때
* 제어 명령은 제자리 회전(선속도=0, 각속도=최대 `pctrl_w_max`)만 내립니다.

### 2.2 RL Controller 모드 (강화학습 기반 메인 자율 주행)
목표 각도가 맞춰진 상태에서, 라이다(LiDAR) 센서 데이터와 ONNX 모델이 정상 작동할 때 동작하는 주력 회피/주행 모드입니다.
* **관측(Observation) 벡터 (28D)**:
  * 라이다 측정값을 24개 방향 섹터로 압축 및 정규화(1=빈공간, 0=장애물).
  * 목표까지의 거리(`dist_n`), 각도 요소(`cos`, `sin`), 진행도(`prog`) 총 4가지 상태값 추가.
* **추론 및 후처리 (EMA & Deadzone)**:
  * 강화학습 모델(`sac_actor.onnx`)에 관측값을 넣어 1차 원시 타겟(Raw Action)을 뽑습니다.
  * **EMA 평활화 (Exponential Moving Average)**: `alpha=0.2`를 적용해 이전 제어값(80%)과 현재 제어값(20%)을 섞어 출력 진동과 급발진을 억제합니다.
  * **Deadzone**: 출력된 각속도가 미세할 경우(`angular_deadzone` 미만) 0으로 강제 보정하여 직진성을 높입니다.
* **PD 제어 추종**:
  * RL 모델이 제시한 목표 속도(Target)에 도달하기 위해 비례-미분(PD) 제어를 적용합니다 (`kp_v=0.5`, `kp_w=1.0`). 불필요한 과증폭을 막기 위해 게인이 조정되었습니다.
* **안전 레이어 (Safety Layer)**:
  * 정면 5개 섹터의 최단거리를 감지하여, 장애물이 가까워지면 속도를 제한합니다.
  * `safety_scale_dist` (0.25m): 이 반경 내에 들어오면 거리에 비례해 선속도를 강제로 줄입니다.
  * `safety_stop_dist` (0.12m): 이 반경 이내로 들어오면 선속도를 0으로 만들어 비상 정지합니다. 최소 0.15m 거리 필터를 적용해 노이즈로 인한 불필요한 멈춤을 방지했습니다.

### 2.3 P-Ctrl Fallback 모드 (비상/폴백 제어)
LiDAR 스캔에 실패하거나 RL 모델을 불러오지 못한 경우에 자동으로 전환되는 예비 모드입니다.
* 장애물 회피 능력은 없으나, 단순 비례(P) 제어를 통해 목표 지점까지 직진 및 조향을 수행합니다.
* 거리에 비례해 속도를 올리고(`pctrl_v_max` 제한), 각도 차이에 비례해 조향(`pctrl_w_max` 제한)합니다.

## 3. 도착 판정 및 파라미터 관리
* **도착 판정 (Goal Tolerance)**: 로봇과 목표 간 거리가 `goal_tolerance` (0.15m) 이하가 되면 도착으로 간주하고, 모터를 정지시키며 주행 루프를 대기 상태(Idle)로 되돌립니다.
* **하드코딩 제거**: 알고리즘 내 사용되던 제어 임계값, 안전거리, PD 게인 등의 매직 넘버가 모두 `robot_app.cpp` 코드 내부에서 제거되었으며, `rl_config.yaml`을 통해 런타임에 유연하게 설정 가능하도록 통합 관리되고 있습니다.
